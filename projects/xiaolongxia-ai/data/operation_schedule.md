# 运营工作计划

## 每日时间表（东八区）

### 🌅 早班 09:00-10:00
- 检查昨晚发布笔记的数据
- 记录数据到 note_stats.json
- 如有爆款（观看>500），立即@老板

### 🌞 中班 13:00-14:00
- 追踪上午发布笔记的表现
- 监控评论区，记录用户反馈
- 如有重要评论（质疑/好评/提问），@老板

### 🌙 晚班 21:00-22:00
- 汇总当天所有笔记数据
- 更新 note_stats.json
- 做简单分析：表现最好/最差的笔记
- 准备第二天的数据报告

## 每周工作（周日晚21:00）

### 📊 周报生成
- 汇总本周数据：总阅读/点赞/评论/涨粉
- 找出表现最佳的3篇笔记
- 分析：什么类型内容表现好？
- 下周建议：选题方向/发布时间/内容形式
- 发送到群里，@老板

### 🔍 竞品分析
- 观察3-5个同类账号
- 记录他们的爆款内容
- 分析可借鉴的点

## 每月工作（月底）

### 📈 月度总结
- 生成月报
- 分析增长趋势
- 识别爆款模式
- 调整下月策略

## 数据文件更新规则

### note_stats.json
- 每次更新笔记数据时，记录 `updated_at` 时间戳
- 观看量变化超过50%时，记录到 `changes` 数组
- 评论内容需要人工判断价值，重要评论复制到 `comments_analysis`

### 分析报告
- 位置：`/root/clawd/projects/xiaolongxia-ai/data/analysis/`
- 命名：`weekly_YYYY-MM-DD.md`
- 月报：`monthly_YYYY-MM.md`

## 改进需求提交流程

当发现以下情况时，@老板：

1. **内容策略问题**
   - 某类内容连续3篇表现差（观看<50）
   - 用户评论反馈负面
   - 竞品有新的爆款模式

2. **运营效率问题**
   - 发布时间效果差，需要调整
   - 互动率持续下降
   - 流量异常波动

3. **选品机会**
   - 发现潜在爆款产品
   - 竞品开始卖某产品且效果好
   - 用户评论里有需求

4. **技术问题**
   - 数据记录出错
   - 分析工具需要优化

## 通知优先级

- 🔴 **紧急**：爆款出现/负面舆情 → 立即@老板
- 🟡 **重要**：数据异常/选品机会 → 当天晚上@老板
- 🟢 **常规**：周报/月报 → 按时发送，不需要@老板
