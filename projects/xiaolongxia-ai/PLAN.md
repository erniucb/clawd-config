**一句话**：一个被AI完全接管的小红书号，展示AI能干啥，顺便卖提示词。

## Agent 架构

```
┌─────────────────────────────────────────────────────────┐
│                    小龙虾内容工厂                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐          │
│  │ 研究Agent │ → │ 写作Agent │ → │ 审核推送  │          │
│  │ (Research)│    │ (Writing) │    │ (Review) │          │
│  └──────────┘    └──────────┘    └──────────┘          │
│       │               │               │                │
│       ↓               ↓               ↓                │
│  扫描AI热点      写小红书文案     发到Telegram          │
│  选3个选题       +emoji+标签      等boss确认            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Agent 1: 研究Agent (每天早上自动运行)
- **输入**：AI领域热点（微博/知乎/小红书/推特）
- **处理**：筛选适合小红书的话题
- **输出**：3个选题 + 素材链接 + 爆款理由

### Agent 2: 写作Agent (手动触发或自动)
- **输入**：选题 + 小龙虾人设
- **处理**：写小红书风格文案
- **输出**：完整笔记（标题+正文+emoji+标签）

### Agent 3: 审核推送
- **输入**：完成的笔记
- **处理**：格式化推送到Telegram
- **输出**：你审核 → 手动复制发布

## 风险控制

1. **平台风险**
   - 不频繁发营销内容
   - 60%干货 + 30%互动 + 10%软广
   - 避免敏感词（微信、加群、私信等）

2. **内容风险**
   - AI生成内容标注"AI创作"
   - 避免虚假宣传
   - 翻车也要诚实展示

3. **封号风险**
   - 发布环节人工操作
   - 不用自动化发布工具
   - 频率控制（每天1-2篇）
