## 教学文章写作风格分析

### 1. 结构设计（分级进阶）

```
1. 什么是OpenClaw
2. 新手阶段：基础入门
3. 中级阶段：实用技能
4. 高级阶段：高级应用
5. 实践任务清单
6. 常见问题解答
7. 学习资源
```

**要点**：
- 新手→中级→高级，循序渐进
- 每个阶段难度递增
- 用户可以按自己的水平跳转

### 2. 每个模块的结构

**固定模板**：
```
## 模块标题

**说明文字**（1-2段）

bash代码块（命令）

✅ 完成标准：xxx

实践任务（可选）：
1. xxx
2. xxx

✅ 完成标准：xxx
```

**要点**：
- 每个模块都有"完成标准"
- 命令用代码块展示，可复制
- 实践任务具体可操作

### 3. 完成标准设计

**❌ 差的完成标准**：
- "学会使用OpenClaw"
- "掌握Skills安装"

**✅ 好的完成标准**：
- "运行openclaw --version能正常显示信息"
- "成功安装并使用至少3个Skills"
- "OpenClaw能够成功并行执行多个子任务"

**要点**：
- 具体可验证
- 用户能自己判断是否完成

### 4. 代码展示

**bash命令**：
```bash
# 一键安装
curl -fsSL https://openclaw.ai/install.sh | bash

# 验证安装
openclaw --version
```

**要点**：
- 每个命令都有注释
- 一行一功能
- 可直接复制运行

### 5. 实践任务设计

**每个阶段都有实践任务**：
```
实践任务：配置子Agent
1. 尝试创建一个简单的并行任务
2. 观察OpenClaw如何协调

✅ 完成标准：OpenClaw能够成功并行执行多个子任务
```

**要点**：
- 任务具体（"创建一个简单的并行任务"）
- 有观察点（"观察如何协调"）
- 有完成标准（"成功执行多个子任务"）

### 6. 资源汇总

**文末提供**：
- 官方资源（网站/GitHub/文档）
- 视频教程（带时长）
- 文字教程（难度标注）
- 社区链接（Reddit/Discord）

**要点**：
- 分类清晰
- 带预估时长（1小时/30分钟）
- 难度标注（新手/进阶）

### 7. 避免AI味的技巧

**❌ AI味**：
- "首先，我们需要配置..."
- "接下来，让我们来学习..."
- "总而言之，OpenClaw是一款..."

**✅ 这篇文章的写法**：
- 直接上命令（"打开终端，输入以下内容"）
- 不废话（"✅ 完成标准：xxx"）
- 偶尔口语（"祝你在OpenClaw的探索之旅中收获满满！🦞✨"）

## 供韩寒学习要点

### ✅ 要学的（教学类）
1. **分级进阶**：新手→中级→高级
2. **完成标准**：每个模块都有可验证的标准
3. **代码块**：命令可复制，有注释
4. **实践任务**：具体可操作
5. **资源汇总**：分类清晰，带时长

### ❌ 避免的
1. 废话太多（"首先让我们来了解..."）
2. 完成标准模糊（"学会使用"）
3. 命令不可复制（截图或描述）
