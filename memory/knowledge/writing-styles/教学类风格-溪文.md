# 教学类文章风格要点

**来源**: 溪文的AI炼金术 - OpenClaw完全教程

---

## 核心结构

```
新手阶段 → 中级阶段 → 高级阶段 → 实践任务 → 常见问题 → 学习资源
```

---

## ✅ 必须遵守

### 1. 每个模块都有完成标准
```
❌ "学会使用OpenClaw"
✅ "运行 openclaw --version 能正常显示信息"
```

### 2. 代码块可复制
```bash
# 注释说明
命令

# 注释说明
命令
```

### 3. 实践任务具体可操作
```
实践任务：配置子Agent
1. 创建一个简单的并行任务
2. 观察执行过程

✅ 完成标准：成功并行执行多个子任务
```

### 4. 资源汇总分类清晰
- 官方资源
- 视频教程（带时长）
- 社区链接

---

## ❌ 避免的

1. 废话太多（"首先让我们来了解..."）
2. 完成标准模糊（"掌握使用方法"）
3. 命令不可复制（用截图或描述）
4. 没有实践任务

---

## 📝 模板

```markdown
## 模块标题

说明（1-2段）

bash代码块

✅ 完成标准：xxx

实践任务：
1. xxx
2. xxx

✅ 完成标准：xxx
```
